# Security Policy

## Supported versions

| Version | Supported |
| ------- | --------- |
| 1.x     | ✅ active  |
| < 1.0   | ❌         |

## Reporting a vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, use one of these private channels:

- **Email**: security@kingegoat.dev (or open a GitHub Security Advisory)
- **GitHub Security Advisories**: https://github.com/kingegoat/devcraft-studio/security/advisories/new

Include in your report:

- Type of issue (e.g. SQL injection, XSS, SSRF, dependency confusion)
- Full path / URL where the issue exists
- Steps to reproduce or a proof-of-concept
- Impact assessment if known
- Your name / handle for credit (optional)

## Response timeline

| Step | Time |
| ---- | ---- |
| Acknowledge receipt | within 72 hours |
| Triage and assess severity | within 1 week |
| Patch for critical issues | within 1 week |
| Patch for high / medium | within 30 days |
| Public disclosure | after fix is released |

## Recognition

We credit reporters in the release notes (unless you ask to stay anonymous).
